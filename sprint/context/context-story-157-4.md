# Story 157-4 Context

## Title
[ENGINE] factions on trope/seed + trope gate + seed-deck draw (Seams 3 & 4)

## Metadata
- **Story ID:** 157-4
- **Type:** story
- **Points:** 5
- **Priority:** p2
- **Workflow:** tdd
- **Repo:** server
- **Epic:** Faction/zone-scoped content eligibility

## Problem
Multi-region worlds leak content across zones. Tropes and seeds are drawn world-wide with no region predicate:
- `trope_tick._gate_activations()` has a comment noting region filtering is "a future seam"
- `SeedDeck` is keyed by `(genre, world, session)`, never region

A Houyhnhnm-faction seed or trope can activate on the Lilliput shore.

## Technical Approach

### Seam 3 — Trope Activation Gate
File: `game/trope_tick.py`, `_gate_activations()` 

Add `factions: list[str]` field to `TropeDefinition` and `SeedTrope` models in `genre/models/tropes.py` (same pattern as bestiary entries in 157-2).

At trope eligibility check, filter dormant trope candidates by:
```
is_eligible(trope.factions, active_factions(perspective=None), zoned)
```
where `active_factions(perspective=None)` returns the **union** of all seated PCs' region `controlled_by` (party-global, split-party safe).

A trope tagged `factions: ["the_houyhnhnm_assembly"]` is dropped as a candidate on the Lilliput shore.

### Seam 4 — Seed-Deck Draw
Files: `game/seed_deck.py` `draw()`, `game/seed_tick.py` `draw_engaged_seed()`

Filter seed-trope candidates **before the draw** by:
```
is_eligible(seed.factions, active_factions(perspective=None), zoned)
```
The deterministic shuffle and resume-safety are untouched — only the candidate set is filtered, not the ordering.

A Houyhnhnm seed is never dealt when seated in Lilliput.

### Shared Predicate & Helpers
Reuse the `zone_eligibility` module created in 157-2:
- `is_eligible(content_factions, active_factions, *, zoned)` — runtime permissive (strictness is the validator's job in 157-7)
- `active_factions(snapshot, pack, *, perspective=None)` — split-party safe union for party-global seams

No new resolution logic — use the existing `region_for()` → `controlled_by` path.

## Acceptance Criteria

### Trope Gate (Seam 3)
1. **AC-3.1:** `TropeDefinition` and `SeedTrope` models carry `factions: list[str] = Field(default_factory=list)`
2. **AC-3.2:** `_gate_activations()` filters candidates using `is_eligible(trope.factions, active_factions(...), zoned)`
3. **AC-3.3:** A trope tagged with a faction not matching the active party region is excluded from candidate list
4. **AC-3.4:** An untagged trope (permissive) or world-global trope (`factions: ["*"]`) passes the gate in any region
5. **AC-3.5:** Split-party (PCs in different regions) receives tropes tagged for the union of their zones
6. **AC-3.6:** OTEL span `zone_eligibility.filtered` fires when a trope is excluded by faction mismatch, with subsystem="trope", content_id, content_factions, active_factions, region

### Seed Draw (Seam 4)
7. **AC-4.1:** `draw()` and `draw_engaged_seed()` filter candidate seeds by `is_eligible(seed.factions, active_factions(...), zoned)` **before** selecting from the shuffled deck
8. **AC-4.2:** A seed tagged with a faction not matching the active party region is excluded before draw
9. **AC-4.3:** Seed deck ordering (determinism, resume-safety) is preserved — only candidates are filtered
10. **AC-4.4:** An untagged seed or world-global seed passes draw filters
11. **AC-4.5:** Split-party seeds are drawn from the union of all seated PCs' zones
12. **AC-4.6:** OTEL span `zone_eligibility.filtered` fires on seed exclusion, with subsystem="seed", content_id, content_factions, active_factions, region

### Wiring & Testing
13. **AC-5.1:** Unit tests for `is_eligible()` covering: unzoned, unresolvable, untagged-permissive, tagged-match, tagged-mismatch, `"*"` sentinel, split-party union
14. **AC-5.2:** One wiring test per seam: synthetic zoned world + Houyhnhnm trope/seed + party in Lilliput → candidates excluded + `zone_eligibility.filtered` span fires
15. **AC-5.3:** When party moves to Houyhnhnm-land, same trope/seed becomes eligible and **is** included in candidate list
16. **AC-5.4:** All acceptance criteria verify via OTEL spans, not source-grep wiring

## Scope
