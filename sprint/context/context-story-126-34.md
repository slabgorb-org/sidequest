# Story 126-34 Context

## Title
[OTEL] Keep test-run sessions out of the live GM dashboard

## Problem
Test-run sessions (test-*, tool-test) accumulate in the live WatcherHub and pollute the GM dashboard's liveSessions picker, pushing real sessions off-screen. 41 stale test-* sessions linger in memory under ADR-122's never-evict policy; only a server restart clears them. The root cause: test runs are pointed at the same port as the live server, so they register with the operator's live WatcherHub. Additionally, session-list and state polling calls bump last_activity_ts, causing auto-follow to chase phantom "activity" in idle test sessions. GM panel lie-detector visibility is obscured because all activity types (real turns, polling, infra spans) mix without distinction.

## Technical Approach
Four-part isolation strategy (short-term UX + long-term architecture):

1. **UI Filtering (Short-term UX fix)**
   - Drop test-*/tool-test sessions from liveSessions picker (useLiveSource.ts:347-355)
   - Drop test sessions from State-tab debugState display
   - Prevents clutter without architectural change

2. **Activity Timestamp Decoupling (Fix auto-follow)**
   - Stop bumping last_activity_ts on session-list/state polling calls
   - Only genuine player/narrator turn actions advance the timestamp
   - Ensures auto-follow focuses on genuinely-active sessions

3. **OTEL Span Tagging (GM panel clarity)**
   - Tag infra/test spans in WatcherHub session activity
   - Session-pin filter in GM dashboard can then exclude test-session spans
   - Enables lie-detector verification: real game vs. noise

4. **Architectural Fix (Root elimination — deferred)**
   - Point test runs at separate port / no-watcher mode so they never reach operator's live WatcherHub
   - Prevents test-session creation in the first place
   - Requires test runner config refactor + server startup variance

## Scope
- **In scope:** UI filtering, activity timestamp logic, OTEL tagging for test sessions, documentation of architectural fix design
- **Out of scope:** Implementation of the architectural fix itself (separate port / --no-watcher mode); that is deferred as tech-debt
