# Story 124-2 Context

## Title
Dependency-aware flame chart — emit + render span hierarchy

## Overview
The Tufte design renders the flame chart with dependency nesting: spans nested caller ▸ callee (containment = depth), faded container spans vs solid leaf work (Tufte layering), and the critical span outlined in accent. 

BLOCKER at redesign time: TurnSpan is FLAT (name, component, start_ms, duration_ms) — no parent/depth/leaf. The shipped FlameChart is therefore an honest time-proportional Gantt (one row per span, slowest outlined), NOT nested. 

To get nesting the server must emit span hierarchy (parent id or depth, and a leaf flag) on the turn_complete spans payload; charts/FlameChart.tsx then lays rows out by depth with container-vs-leaf opacity layering and keeps the accent outline on the critical span.

- TurnSpan model in `sidequest/game/turn.py` or similar needs to emit hierarchy data
- turn_complete event payload needs to include parent_id/depth + leaf flag for each span
- Must preserve backward compatibility for clients that don't understand the hierarchy (fallback to flat Gantt)

### UI Side (sidequest-ui)
- FlameChart component in `src/components/Dashboard/` needs to accept hierarchy data
- Render logic: nest spans by depth, use opacity layering (faded containers, solid leaves)
- Keep critical/bottleneck span outlined in accent color per existing design
- Graceful degradation: if hierarchy data missing, render flat Gantt as before
