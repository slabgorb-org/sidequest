## Overview
Companion-seat brain telemetry surfaces in the SideQuest Inspector.

Long-running companion NPCs that travel with the party — pets, henchmen (peer), hirelings (the seat_core Role enum: PET/PEER/HIRELING) — are driven by the understudy seat_core brain, one LLM decision per turn (claude_p/anthropic/ollama). That brain work is invisible today: claude_p discards the `claude -p` JSON envelope's usage/total_cost_usd (hardcodes input/output tokens=0, claude_p_model.py:60), and the companion run loop records no duration, outcome, or span. Goal: surface per-decision brain telemetry (duration, model, tokens, cost, YIELD-vs-act outcome) for the WHOLE entourage — keyed by session slug + seat + role — in a dedicated new Inspector tab, giving the dev the same lie-detector visibility the GM panel gives for the server narrator. Found during epic-160 dogfood (2026-06-27, MP beneath_sunden, owl Tolliver).

DESIGN RESOLVED (161-1) — Approach C: understudy self-reports each decision to the server's EXISTING /internal/watcher/emit bridge (ADR-131, the door the daemon already uses via watcher_bridge.py); it rides WatcherHub -> /ws/watcher -> Inspector and persists to turn_telemetry. Rejected: (A) reading the native claude -p :55801 OTEL — that collector is BikeRack's pf.frame.app, populated only by an ambient CLAUDE_CODE_ENABLE_TELEMETRY the dev harness happens to set (a silent-fallback trap), and native subprocess spans cannot carry session/seat/role/YIELD-vs-act semantics; (B) a new SideQuest OTLP receiver — new infra to catch the same unattributable spans. Because each process reports its own decisions in-band, the "coordinate the two claude -p processes" problem never arises.

Two corrections to the original breakdown: (1) the native :55801 ingestion path is DROPPED; (2) the render surface is the Inspector/Dashboard tab system (DashboardTabs + DashboardApp index render + useLiveSource reducer), NOT the game-board widgetRegistry + MobileTabView (those are player-facing tabs). Server footprint is small: a session_slug + severity passthrough on /internal/watcher/emit + publish_event. Full design: sprint/context/context-story-161-1.md.
