## Problem
From the 124-1 Reviewer audit. TokenBarChart.tsx:143 — the cache-hit-rate dot tooltip builds its turn label from the post-`filter` index (`i+1`) of the hitPoints array, not the turn's real `turnIndex`. The dot's x-position (p.x, captured pre-filter) and the aggregate cold-miss count are correct; only the hover label is wrong, and only when one or more null (non-SDK) turns are interspersed before a known turn. Fix: carry `turnIndex` into each hitPoints object and use it in the <title>. Add a regression test: a series like [null, cold] must show the cold dot's tooltip as the SECOND turn, not 'T1'. Affects sidequest-ui/src/components/Dashboard/charts/TokenBarChart.tsx + telemetryAdapter-cache-join/TokenBarChart-cache-split tests.
