## Problem
Found in 2-player MP playtest of beneath_sunden (2026-06-23, session 2026-06-23-beneath_sunden-mp-e88e04d6), exercising feature #1062 (per-expansion quests, shipped same day). The feature's happy path works end-to-end (mint -> project -> complete; lie-detector PASS; exp3 + exp10 completed on the reach_deep beat). The problem: every player-visible quest reads "Below the Black Water" (drowned_cavern theme) across a 10-expansion descent, even though the world ships 5 themes each with a distinct quest_template. The operator (Keith) flagged it: "we should have a few."

DB-grounded (psql dungeon_map for the session): region themes DO diversify with depth — winding_catacomb regions from exp006, bone_crypt from exp008 — so the dungeon is NOT thematically frozen. The QUEST mechanism is the culprit.

Root causes:
1. Quest binds the DEEPEST region's theme (expansion_quest.select_signature -> _deepest). drowned_cavern's depth_band {0,60} is the widest, so it keeps winning the deepest slot even at depth 42-52 (DB: exp005.r3@42.2 and exp010.r0@52.8 both drowned while sibling regions were winding/bone). Region variety appears by exp6; quest variety never does.
2. Shallow zone is genuinely single-theme: at depth_score 0-19 the eligible theme_pool is exactly [drowned_cavern] (themes.themes_for_depth), so rng.choice has one element -- the opening is monotone by construction.
3. Sparse projection: reconcile_dungeon_quests_into_log only projects threads whose expansion_id == the to_region's expansion. The multi-expansion-per-turn jumps pass THROUGH exp6-9 (the diverse-theme expansions) without landing in them, so their quests never reach quest_log. Player-visible quests were exp1/3/5/10 -- all drowned-deepest.
4. Static title: drowned_cavern.yaml quest_template.title is the literal "Below the Black Water" with no {region}/{depth}/ordinal slot in expansion_quest._fill, so same-theme repeats are byte-identical.

Repro: MP beneath_sunden, descend ~6 turns to exp010, open Quests tab -> 4 entries all "Below the Black Water" (2 active, 2 completed). See ~/Projects/sq-playtest-pingpong.md (2026-06-23 session, the high-priority pile-up entry) + playtest-shots/108-p2-quest-pileup-3-identical.png.

Repos: content (theme depth_bands + title text) + server (expansion_quest binding/projection logic). The exact depth_band values are a crunch/pacing call -> Keith owns them. Related but separately filed: stale-active quests never GC; multi-expansion-per-turn pacing; theme/depth-band incoherence (bone_crypt region at depth_score 8.2).
