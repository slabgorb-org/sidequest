## Problem
ADR-140 layering for chargen kits. PROBLEM: caverns (and likely road_warrior) verbatim sweeps move no-SRD-analog dungeon items to the world tier, but the genre equipment_tables.yaml class_tables + guaranteed_grants reference them — so genre kits must go SRD-pure and the world must re-add the flavor. There is no world-tier equipment_tables today. SOLUTION (Keith Option-B, 2026-06-15): World model gains an optional equipment_tables field; loader loads worlds/<w>/equipment_tables.yaml; CharacterBuilder merges world-over-genre. MERGE SEMANTICS: per-slot APPEND within each class kit (genre slot items first, then the world's), and guaranteed_grants APPEND by kit_id. ADDITIVE: a world that ships no equipment_tables.yaml is a pure no-op (genre kits unchanged) — this is why it lands green on its own. Builder seam: builder.py uses self._equipment_tables (genre); needs the world's merged in (chargen already has world_slug — see chargen_mixin.py:1242 / resolve_inventory precedent). FIXTURE-TESTED in server (synthetic WWN pack + world). Caverns CONTENT split + the ~6 caverns chargen-test updates ride on this in 120-1. Cross-ref: ADR-140 (world owns the catalog), ADR-121 (layered resolution), session .session/120-1-session.md 'Dev note' for the full plan.
