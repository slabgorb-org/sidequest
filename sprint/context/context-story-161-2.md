## Problem
PRODUCE companion-seat brain telemetry (server + understudy), per the 161-1 design (Approach C). (a) seat_core: enrich DecideResult with model + real tokens + cost across backends — parse the `claude -p` JSON envelope's usage/total_cost_usd (currently discarded at claude_p_model.py:60 -> hardcoded zeros); anthropic already reports real tokens (incl. cache), ollama likewise. (b) companion run loop (companion/run.py): time the decide() await, derive outcome (intent_kind + yielded) and degraded/timed_out, assemble a `companion_brain_decide` record tagged with session_slug (defn.game_slug) + seat (defn.name) + role (pet/peer/hireling) + species, and emit it via a NEW understudy->server bridge module mirroring the daemon's watcher_bridge.py (stdlib POST, 2s timeout, failures logged WARNING and swallowed for-the-turn-only — never break a companion's move). (c) server: add optional session_slug + severity to WatcherEmitPayload and an explicit session_slug/severity override to publish_event (override beats the ContextVar; default None keeps every existing in-process caller and the daemon unchanged). Component="companion_brain", event_type="companion_brain_decide". Covers all companion roles, not just the owl pet.

## Acceptance Criteria
- seat_core DecideResult carries model, input/output/cache tokens, and cost_usd; claude_p parses these from the JSON envelope (no more hardcoded zeros); anthropic and ollama fill real values
- companion run loop emits one companion_brain_decide event per decision with fields: seat, role, species, owner, outcome, intent_kind, degraded, timed_out, backend, model, duration_ms, tokens, cost_usd — tagged with session_slug
- Timed-out or errored decisions still emit, with degraded=true and severity=warning (silence is not acceptable for a lie-detector)
- server /internal/watcher/emit accepts optional session_slug + severity and forwards to publish_event; existing callers and the daemon bridge are unaffected (default None preserves ContextVar behavior)
- emit failures are logged at WARNING and never break the companion's turn (fail-loud-non-fatal, mirrors watcher_bridge.py)
- wiring test: a fake-brain companion decision produces a companion_brain_decide watcher event on the hub with correct session_slug + seat + role (behavior/OTEL assertion, not source-text grep)
