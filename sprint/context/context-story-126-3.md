## Problem
WORKING-AS-DESIGNED per ADR-144 (reclassified from [FATE-BUG]). The 114-10 Fate-gear migration (#472) deleted inventory.yaml for the four Fate packs; Fate has no carried inventory and no economy (game/ruleset/fate_gear.py + docs/superpowers/specs/2026-06-15-fate-gear-model-design.md: a ruleset:fate pack shipping inventory.yaml is a defined HARD ERROR). Gear dissolves into aspects via source_gear. The gap is pure UX: a Fate player who opens 'Inventory' sees an empty native panel (items:[], gold:0) and never learns their gear became aspects. DECISION (Keith, 2026-06-17): option (a) — HIDE the native Inventory tab for ruleset:fate packs (option (b), a read-only FatePanel Gear view, was declined). UI-ONLY change (no server work despite the epic's server repo tag) — do NOT populate inventory.items/gold (that re-introduces the carried inventory ADR-144 deliberately deleted). GOTCHA: a GameBoard dock tab is registered in TWO places — widgetRegistry/GameBoard AND MobileTabView's own TABS array; the jsdom wiring tests render the mobile path, so the conditional-hide must cover BOTH or the tab-reachability assertion will mismatch. Source: sq-playtest-pingpong [FATE-BUG] inventory; FIXER ruling (Dev/Hephaestus).
