## Problem

Story 108-1 removed the native riders (fleeting tags, dial advances, Brace action) from the WN round, but `run_wn_round()` (sidequest-server/dispatch/wn_round.py:374) and the dice resolution path (dispatch/dice.py:407) still resolve the player's committed action by looking up `commit.beat_id` in `cdef.beats` and feeding that beat to `_apply_committed_player_beat()` for the WN damage math.

With native beats stripped in 108-3, `cdef.beats == []` and EVERY WWN combat commit raises:
```
DiceDispatchError: unknown beat_id ... available: []
```

This is a total combat outage (78 e2e failures across caverns_and_claudes, heavy_metal, elemental_harmony, beneath_sunden, barsoom).

**Root cause:** The runtime (run_wn_round + dice.py) still requires cdef.beats to resolve the action, even though story 108-1 was meant to decouple the WN math from the native beat scaffolding. The runtime half was never implemented.

## Technical Approach

Under a `WithoutNumberRulesetModule` binding, `run_wn_round()` and `dice.py` must synthesize the WN action set **independent of cdef.beats**, modeling it on the existing `is_item_use_beat()` transient-beat synthesis (the "Drink potion" beat that exists in the WN action space but is NOT in cdef.beats).

**Key design points:**

1. **Transient beat synthesis:** Create a transient beat object for each WN action (attack, move, item-use, cast) when resolving a committed action under a WN binding, bypassing the cdef.beats lookup entirely.

2. **WN math preservation:** Keep the full WN resolution math intact:
   - d20 + hit modifier vs AC
   - Weapon dice from actor's inventory
   - Shock mechanical (if applicable)
   - Saving throws
   - Ablative HP damage resolution (ADR-114)

3. **Native guard:** Native packs MUST keep requiring beats via an `isinstance(WithoutNumberRulesetModule)` gate. A native pack with zero beats must still raise an error at load time (108-7 enforces this).

4. **OTEL lie detector:** Emit `wn.native_scaffolding_suppressed` proving the native riders (fleeting tags, dial advances, Brace) did NOT fire under the WN binding.

**Seams to modify:**
- `dispatch/dice.py` ~671-676 (beat resolution entry point)
- `run_wn_round()` beat-resolution path (wn_round.py:374)
- `game/ruleset/without_number.py` (transient beat factory / synthesis helper)

**Dependencies:**
- Unblocks 108-3 (content de-nativize, 78 test failures)
- References: ADR-143, ADR-142, ADR-117 (RulesetModule), ADR-139 (Other invariant), ADR-114 (ablative HP)

## Acceptance Criteria

1. **Zero-beat WWN combat defs resolve without DiceDispatchError:**
   - A WWN combat def with `beats: []` can resolve a committed WN action (attack/move/item-use/cast)
   - No lookup error on `cdef.beats`
   - Combat completes, damage resolves, and HP is deducted correctly

2. **WN action set is synthesized under WN binding only:**
   - Transient beats (attack/move/item-use/cast) are created only when `isinstance(ruleset, WithoutNumberRulesetModule)`
   - Native packs with zero beats still raise an error (108-7 gate honored)
   - No regression to native pack behavior

3. **WN math is preserved end-to-end:**
   - d20 + hit vs AC resolution correct
   - Weapon dice applied from actor inventory
   - Shock/saves/ablative damage resolved per WN SRD

4. **OTEL lie-detection working:**
   - `wn.native_scaffolding_suppressed` is emitted and visible in GM panel
   - Span proves native riders (Opening tag, Counter Stance, dials, Brace) did NOT fire
   - GM panel verifies the WN path is active

5. **No regression to integration:**
