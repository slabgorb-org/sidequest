## Problem
Found by the first real-server companion playtest (2026-06-26, Architect / sq-playtest). Donut connects, seats, and its claude_p/sonnet brain fires in-character — but STALLS at character creation. Root cause: companion/run.py::_chargen_choice forwards the brain's ACT prose verbatim as the scene choice, and companion/protocol.py::chargen_choice_frame sends {phase:'scene', choice:<prose>}. The server (sidequest-server/sidequest/server/websocket_handlers/chargen_mixin.py::_chargen_scene) resolves a choice by (1) int(choice_str) as a 1-based index, else (2) EXACT case-insensitive match against choice.label, else falls through to builder.apply_freeform() — wrong for a select scene (class pick Warrior/Expert/Mage). A whole prose paragraph matches neither numeric nor exact-label, so chargen never advances and the entire companion lifecycle (play, dice, bond/pet-widening) is blocked behind it. The scripted full-loop fixture (tests/companion/test_full_loop.py) missed this because it accepts whatever the companion sends and does not enforce the real server's choice contract. Detail: ~/Projects/sq-playtest-pingpong.md (2026-06-26 companion playtest, BLOCKING entry).

## Acceptance Criteria
