## Problem
The 2026-06-14 audit cross-checked pack_schema.yaml against the real genre-pack loader (sidequest-server/sidequest/genre/loader.py). They have drifted apart in BOTH directions, so the pack validator can neither flag dead files nor recognize live ones.

SCHEMA LISTS as genre extensions, but loader DROPPED (dead):
- openings.yaml — loader.py:1831 sets openings=[] ; openings are world-tier only now
- powers.yaml — fully UNREFERENCED in server + daemon
- weather.yaml — repointed to world_dir (Epic 74); pack-root file no longer consulted

LOADER READS, but schema OMITS (live mechanics, currently invisible to the validator):
- skills.yaml (loader.py:1892), spells_wwn.yaml (:699/1920+), foci.yaml (:1877),
  bestiary.yaml (:1950), backgrounds.yaml (:1861), witnessed_acts.yaml (:1973),
  mutations.yaml (:1985), disciplines_psionic.yaml (:1932)

ROOT CAUSE: the loader reads a fixed hard-coded filename allowlist (no glob at pack root), so any unknown YAML dropped at a pack root is silently ignored. The schema's extensions[] list is hand-maintained and went stale against that allowlist.

SCOPE: make pack_schema.yaml the source of truth for the loader allowlist (or generate/validate one against the other) so the two cannot silently diverge again. Decide whether validator should WARN on unknown pack-root YAML (currently silently ignored — a No-Silent-Fallbacks gap).
