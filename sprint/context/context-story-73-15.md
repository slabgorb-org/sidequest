## Problem
Architect finding (2026-06-04, ADR-117 ruleset routing dig). orchestrator.py:3844/3992 hands the narrator default_registry.tool_definitions() — ALL registered tools, unfiltered by bound ruleset. The WWN/CWN-only tools (commit_effort/long_rest/veterans_luck=wwn; adjust_system_strain/stabilize_mortal_injury=cwn) are therefore visible to the narrator on EVERY pack, including native packs. They self-guard fail-loud: tool description says 'WWN-only…' and the handler raises ValueError if pack.rules.ruleset != the required slug (e.g. commit_effort.py:96). So this is NOT a correctness bug (no silent fallback) — it is a tightening: the narrator wastes tool-budget on, and can mis-attempt, tools irrelevant to the bound ruleset. Tighten by filtering tool_definitions to the bound ruleset at assembly time. NOTE: SM may re-home — no dedicated ruleset epic exists; parked in 73 because the gated tools are combat/lethality tools.
