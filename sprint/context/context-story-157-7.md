## Problem

Multi-region worlds leak content across zones because the **runtime** `is_eligible()` is **permissive** (untagged content is permitted to allow staged rollout). Engine seams (157-2/3/4) and content tagging (157-5/6) are complete; now we lock the door with a **strict load-time validator** that rejects any untagged pooled content in a zoned world at pack-load time.

The validator must:
1. Detect zoned worlds (any region with `controlled_by` set)
2. For every pooled, home-less authored item (bestiary/trope/seed_trope):
   - Assert `factions` is non-empty (no silent untagged pass)
   - Assert every faction value is either `"*"` or a real `controlled_by` slug in that world's cartography (catch typos)
3. Raise `GenreLoadError` on any violation, naming the offending item IDs and world
4. Emit OTEL `zone_eligibility.validator_failure` on rejection (so forensics can see what got caught)
5. Do NOT exempt authored cartography NPCs (zone derived, not tagged) or runtime walk-ons (stamped on activation)

## Technical Approach

Per design spec (docs/superpowers/specs/2026-06-20-faction-zone-content-eligibility-design.md §"Strict load validator + error handling"):

1. Add validator to `sidequest/genre/loader.py` after `world_is_zoned()` computed and AFTER trope inheritance resolved (so extends-tropes carry inherited factions).
2. Check world is zoned; if False, skip validator (unzoned worlds have no factions).
3. For each of: bestiary entries, trope definitions, seed tropes in the loaded world:
   - If `world_is_zoned`: assert non-empty `factions`, assert all values in {`"*"`} ∪ {real `controlled_by` slugs}
4. Collect failures; if any, raise `GenreLoadError(offender_ids + world_name)`.
5. Exempt: cartography NPCs (zone derived via region), walk-ons (stamped on activation).
6. Emit OTEL `zone_eligibility.validator_failure` span per rejected item (lie-detector: GM panel can verify rejections happened).

## Scope

- **In scope:** Strict load-time validator (`genre/loader.py`), GenreLoadError on violations, referential integrity checks (typo'd faction slugs), OTEL emission, RED-first unit tests (validator alone), wiring tests (validator fires on real packs).
- **Out of scope:** Runtime predicate `is_eligible()` (157-2), seams (157-2/3/4), content tagging (157-5/6).
- **Sequencing constraint:** lands AFTER 157-6 (all zoned worlds tagged); no content should fail to load in develop once this lands.
