## Technical Approach

**LOAD-BEARING DESIGN CAVEAT:** Freeform pronouns are an INCLUSIVE chargen feature (`builder.py pronouns_allow_freeform`). Do NOT force `Character.pronouns` to a 3-value Literal — that breaks inclusive display. Separate concerns: keep freeform DISPLAY pronouns (what the player chose) and derive/validate a canonical GRAMMATICAL projection for the localizer (`swap_to_second_person` only accepts `{he/him, she/her, they/them}`), or normalize at chargen.

**Root cause (from 158-8 session Reviewer findings):**
- `Character.pronouns: str` (character.py) is unconstrained
- `swap_to_second_person` (pov_swap.py:878) raises `ValueError` for any pronouns outside the canonical set
- Chargen sets `pronouns_allow_freeform=True` (builder.py:2194), allowing freeform strings like "she/they", "any", "xe/xem"
- 158-8 shipped only an in-function fail-open guard in `_apply_pov_swap` that catches non-canonical values AFTER they've reached the emitters layer

**Hardening strategy:**
1. **Validate at the Character model level:** Introduce a validated/constrained field (Literal + validator, or chargen-time normalization) to guarantee the localizer is NEVER handed a non-canonical value from ANY caller
2. **Keep display pronouns separate:** Store the freeform player choice (e.g., "she/they") for UI display; derive/validate a canonical grammatical projection (e.g., "she/her") for the localizer
3. **Complete skip-path instrumentation:** Route ALL `_apply_pov_swap` skip paths (empty, invalid, non-canonical) through `narration.pov_swap_skipped` OTEL spans with distinct `reason` values:
   - `reason='pronouns_empty'` — PC has no pronouns in snapshot
   - `reason='pc_not_in_snapshot'` — PC not found in snapshot at all
   - `reason='text_missing_or_invalid'` — narration text is missing/not a string
   - `reason='unsupported_pronouns'` — non-canonical pronouns (158-8's in-function guard)
4. **Bound the pronouns span attribute:** Truncate/normalize player-supplied values written to the OTEL span ([SEC][LOW] PII/length concern from 158-8 Reviewer)
